Spiral · Prototype 2.4 · Technical Documentation

Physics-grounded plastic LCA & recyclability simulation · Circularity State-Space Framework

1. State Space & Operating Zone

Every material batch is represented as a point in a 3D state space. At every step (Use or Recycle), we track:

s = (E, C, Q) E = cumulative environmental burden [kg CO₂eq/kg] C = cumulative net economic cost [USD/kg] Q = composite material quality index [0, 1]

The operating zone is defined by user constraints: Z = {E ≤ E_max, C ≤ C_max, Q ≥ Q_min}. In the 3D plot, states inside this cuboid are coloured Cyan. Once a trajectory exits the cuboid, the state is considered "Failed" and the markers turn Red, though the simulation continues to trace the ultimate breakdown trajectory. A uniform connecting line traces the entire genealogy.

2. The "Greedy Auto-Selector" Algorithm

If "✨ Greedy Auto-Selector" is chosen, the engine acts as an optimal controller. At the start of each recycle phase, it simulates the outcome of all available recycling technologies. It calculates a "Net Cost Score":

Cost_Score = - (ΔC_recycle + σ_carbon * ΔE_recycle)
  • If valid options exist: The algorithm selects the technology with the highest Cost_Score (i.e. cheapest to run + carbon tax) that successfully clears the Q_min threshold.
  • If no valid options exist: (Because the material is too heavily degraded), it abandons cost constraints and purely selects the technology that yields the maximum possible Q (typically Chemical Recycling) in an attempt to rescue the batch.

3. Root-Cause Variables (Internal State)

The macroscopic Q is derived from three fundamental physical variables tracked per cycle:

s_internal = [ m̄, Xc, Ci ] m̄ = normalised weight-avg mol. weight Mw(n)/Mw₀ ∈ [0,1] Xc = crystallinity fraction ∈ [0, Xc_max] Ci = contamination index (foreign species mass frac) ∈ [0,1] π = purity = 1 − Ci ∈ [0,1]

4. Parameter Editors & Physics Equations

In the Settings tab, three Editors control how the simulation runs. The underlying math uses these exact parameters:

4a. Technology Editor (Recycle Phase)

  • Restoration Efficiency (ρ): How well the technology rebuilds chains.
    Q_restored = Q_before · (1 − ρ) + ρ · m̄^0.4 · π^0.3
  • Purification Efficiency (φ): How much contamination is removed.
    Ci_new = Ci_old · (1 − φ)
  • Energy Burden (ε): Base energy emissions, which scale up if material is dirty or degraded.
    ΔE_recycle = ε / (Q · π) + ΔE_virgin
  • Economic Cost (κ): Base financial cost, scaling with impurity.
    ΔC_recycle = κ / π + ΔC_virgin - Revenue(Q)

4b. Polymer Editor

  • Use-phase scission (β_u): The rate at which chains break due to UV/thermal aging during the product's life. Accelerated by existing contamination.
    m̄_new = m̄_old · exp(−β_u · (1 + Ci))
  • Melt scission (β_r): The rate at which chains break due to sheer stress and heat in the recycling extruder.
    m̄_after_melt = m̄_before_melt · exp(−β_r)

4c. Application Editor (Use Phase)

  • Use-phase E: Direct carbon emissions generated by using/transporting the product. Added to ΔE during the use phase.
  • Collection Cost: The baseline cost to retrieve and sort this type of product. Scales linearly with contamination.
    ΔC_use = Collection_Cost · (1 + Ci)
  • Contamination Growth (k): Determines how fast dirt, labels, or mixed polymers accumulate onto the material per cycle (logistic growth).
    Ci_new = Ci_old + k · (1 − Ci_old)
